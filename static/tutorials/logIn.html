Security

Security API

The Security API will allow the registration of users and application with the OPENi platform.

The registration of applications with OPENi require multiple steps.

  • The developer must register as a user
  • The developer must then login to get a sesion token.
  • A client can then be created using the developers session token.
  • The api_key and secret are then used within the application.

User Registration

  • /users

This endpoint allows you to register new users. The user registration requires a username and a password. If not existent, a new user will be created.

Upon success this call will return an empty response with a 200 status code.

Example:

{
    "username":"developer",
    "password":"password"
}

Login

  • /session

This endpoint allows you to log in existing users. It requires a username and a password. The login will return a session token. The session will expire eventually.

Client Registration

  • /clients

This endpoint allow a developer to register new clients. The client registration requires a name and a description. The session token is passed as Authentication when creating a client.

Upon success this call will return the following response with a 200 status code.

Example:

{
    "name": "OPENi_Android",
    "description": "The Official OPENi Android application"
}

User-Client Authorisation

  • /authorizations

This endpoint allows a User to authorise a Client to act on its behalf and access their cloudlet. The request requires the username and password of the user and the api_key and secret from the client.

Upon success this call will return a session object with a token, as seen below and with a 200 status code.

This token will then be used when making calls to the OPENi Platform.

Example:

{
    "username": "OPENi_user",
    "password": "passowrd",
    "api_key": "b3e51db43d630ea349ac0b4b3dc16e6f",
    "secret": "761dbc37396338c4e3eb00610d923334d9fc282b4f4466d874ddf9dba1cde791"
}
