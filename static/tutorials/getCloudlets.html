Cloudlet and Object Creation

The Cloudlet platform API handles all requests for cloudlets, objects and types for OPENi. Instructions for accessing the Cloudlet API UI from the home page cam be seen below.

Cloudlet Retrieval

The Authorization token received from the security API can be used within the cloudlets : Cloudlet Administration API section with the GET/cloudlets API endpoint to retrieve the cloudlet id for a user to be used when accessing the OPENi platform on their behalf.

At the bottom of the 'GET/cloudlets' section you can then activate it by pressing the "Try it out!" button. This will produce a cloudlet ID.

Type Creation

In order to create an object within the OPENi Platform there must first be a corresponding type upon the platform that an object can conform too.

Type creation is done within the types : Controls OPENi types section using the 'POST/types' endpoint.

This section has some implementation notes about what parameters will be accepted in the body of the POST. An example 'HelloWorld' type can be seen below.

{
    "@context": [
    {
      "@property_name": "hello",
      "@openi_type": "string",
      "@required": true,
      "@multiple": false,
      "@allowed_values": [
        "foo",
        "bar",
        "world"
      ],
      "@context_id": "openi/hello"
    }
  ],
    "@reference": "openi/HelloWorld"
}

This shows an example of all the fields for Types. In this example we can see thwet the "hello" key is required, there cannot be multiple "hello" keys and that the only acceptable values for it are "foo","bar" and "hello".

The responce from the POST should return the id of the newly created type on the OPENi platform.

N.B. Prior to Object creation within a cloudlet the correct Permissions must be set on the OPENi Platform.See tutorial here

Object Creation

Using the HelloWorld type we can create an object within the users cloudlet using the 'POST/objects/{cloudletId}' endpoint.

This section has some implementation notes about what parameters will be accepted in the body of the POST. An example object using the 'HelloWorld' type can be seen below.

{
  "@openi_type": "t_26c91405344400a0b7007f3788f3cf9f-199",
  "@data": {"hello" : "world"}
}

The "@openi_type" key hold the id of the reference type for that object. This field will also accept URLs like so:

{
  "@openi_type": "https://demo.openi-ict.eu/api/v1/objects/c_f0bdae6dd7e375e1095b8e88f87ecec7-90",
  "@data": {"hello" : "world"}
}
