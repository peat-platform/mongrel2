Type and Object Referencing

This section is an example of how Types and Objects within the PEAT platform can themselves contain references to other Types and Objects.

For this example Swagger UI compliant code will be used but note that type creation can also be done via the Java SDK.

This example will user three types.

  • Person: which contain the sub types.
  • Full Name.
  • Address.

N.B. The person type must be created last as it references other types by ID.

The following types must be created upon your PEAT dev environment.

N.B. Keep a record of the IDs returned by the PEAT system as they may be different to the ones used here.

Full Name Type

{
    "@context": [
    {
      "@property_name": "firstname",
      "@property_context": {
        "@id": "",
        "@type": "string"
      }
    },
    {
      "@property_name": "middlename",
      "@property_context": {
        "@id": "",
        "@type": "string"
      }
    },
    {
      "@property_name": "lastname",
      "@property_context": {
        "@id": "",
        "@type": "string"
      }
    }
  ],
    "@reference": "Full_Name"
}

Address Type

{
    "@context": [

      "@property_name": "number",
      "@property_context": {
        "@id": "Number",

      }
    },

      "@property_name": "street",

        "@type": "string"
      }
    },

      "@property_context": {

      }
    },

      "@property_context": {

      }

      "@property_context": {
        "@id": "country",

Person Type

{
    "@context": [
    {
      "@property_name": "name",
      "@property_context": {
        "@id": "Full Name",
        "@type": "t_ff29cd6d53b1e3f4ef4a6ff828d0ddf4-324"
      }
    },
    {
      "@property_name": "email",
      "@property_context": {
        "@id": "Email Address",
        "@type": "string"
      }
    },
    {
      "@property_name": "address",
      "@property_context": {
        "@id": "Full Address",
        "@type": "t_fb651f2d7bd17d930fb45dee08eae660-485"
      }
    }
  ],
    "@reference": "Person"
}

Object Creation

As can be seen above the Person Type references the two other types created during this example.

When a Person type is created the "name" and "address" properties must adhear to their respective types. This can be done in two ways.

  • A type compliant Object literal can be passed in as the Person object is being created, as seen with Full Name below.
  • Or a reference to an already exiting object can be used.

To show an example of both an Address object must first be created using the code below.

N.B. Record the Object ID as it will be different from the one in the example.

Address Object

{
    "@type": "t_fb651f2d7bd17d930fb45dee08eae660-485",
    "@data": {
    "number": "123",
    "street": "Fake St.",
    "city": "Waterford",
    "state": "Co. Waterford",
    "country": "Ireland"
  }
}

Once the Address Object is created The following Person object can be created.

  • Note the "Full Name" Object is not a reference but an object literal.
{
    "@type": "https://192.168.33.10/api/v1/types/t_086a914f04ba528b812006f9e4b5d7a6-378",
    "@data": {
    "name": {
      "firstname": "Test",

      "lastname": "Name"
    },
