Creating PEAT Types

Every object added to a users Cloudlet on the PEAT cloudlet framework must adhere to the rules set out in an PEAT Type object. Each type lists a number of properties and rules around that property which each object is validated against.

PEAT Type Description

PEATType {
    @id (string) //Platform generated Unique identifier for the type,
    @location (string) //Platform generated Unique types URI,
    @context (array[ContextEntry]) //Developer specified array of object property description,
    @reference (string) //Developer specified URL of the refernce type,
    _date_created (Date) //Platform generated date that the Type was created.
}

Context Entry Details

Each context entry must have one of the following entries.

  • @property_name - (String) name of the property in the object.

  • @type - the primitive type that the property will be validated against, the following primitives are allow:

    • boolean
    • int
    • float
    • string
    • url
    • attachment - an PEAT attachment id from the same cloudlet
    • date - string of following format (DD-MM-YYYY)
    • timestamp - sting of following format (YYYY-MM-DD HH:mm:ss.SSS)
    • hexadecimal - a hexadecimal string
    • base64 - a base64 endoded string
    • {peat type id} - the id of another PEAT type. Allows objects to contain other objects.

  • @required - (true or false) specifies if the object can be created with or without this context entry.

  • @multiple - (true or false) specifies if the property is an array or values or a single value.
  • @context_id - A context identifier for the property entry.
  • @allowed_values - this property limits the values that the property can be set to.

Creating a type

The following example creates a profile type. The body of the POST contains a JSON object with the user defined @reference and @context properties. In this case the array of @context properties include a required username of type string, a optional slogan of type string, and an optional attachment with a name of avatar. You may also not that multiple values (or an array of values) are allowed for the slogan property. 

curl --insecure -X POST \
    -H "Accept:application/json" \
    -H "Content-Type: application/json" \
    -d '{
    "@context": [
    {
        "@property_name": "username",
        "@type": "string",
        "@required": true,
        "@multiple": false,
        "@context_id": "https://peat-platform.org/my_app_profile/username"
    },
    {
        "@property_name": "slogan",
        "@type": "string",
        "@required": false,
        "@multiple": true,
        "@context_id": "https://peat-platform.org/my_app_profile/username"
    },
    {
        "@property_name": "avatar",
        "@type": "attachment",
        "@required": false,
        "@multiple": false,
        "@context_id": "https://peat-platform.org/my_app_profile/avatar"
    }
    ],
    "@reference": "https://peat-platform.org/my_app_profile"
    }' \
    https://dev.peat-platform.org:443/api/v1/types

The response is a JSON object containing the newly create type's id.

{
    "id":"t_f3bfd0b665a3e32b1d3c9fe1927717d6-511"
}

Creating an object of that type

curl --insecure -X POST \
    -H "Accept:application/json" \
    -H "Content-Type: application/json" \
    -d '{
            "@type": "t_f3bfd0b665a3e32b1d3c9fe1927717d6-511",
            "@data": {
            "username" : "bob",
            "slogan" : ["Best slogan last week", "Best Slogan ever."],
            "avatar" : "a0e34cdf-6424-480d-b8cc-8350a9ce4eb8"
        }
    }' \
    https://dev.peat-platform.org:443/api/v1/objects/c_897b0ef002da79321dcb0d681cb473d0

The response is a JSON object containing the newly create objects's id.

{
    "id":"00d2c1c6-f440-48af-b774-136ea5f09f87"
}
