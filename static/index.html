OPENi

Open-Source, Web-Based, Framework for Integrating Applications with Cloud-based Services and Personal Cloudlets.

What is OPENi?

To developers OPENi is a fantastic service that simplifies the creation of mobile application by managing the applications interaction with a multitude of cloud based services through the Graph API.

To end users OPENi is a cloud based service that securely stores their personal data. Furthermore it's privacy control mechanisms gives them full control over how and with whome they data is shared.

What is the Graph API?

Graph API is the main interface of interaction with OPENi. After you have got the token from OPENi authentication, you can use the Graph API methods to access resources from OPENi Cloudlets. By following the basic rules of the Graph API, you can call resources coming from the user Cloudlet, the user Cloud Based Services (e.g. Facebook, Foursquare, Twitter etc.) or combine results from both directions. The documentation of the Graph API is available here (/api-docs/?api_framework=true) and the description of how the Graph API is structured is here (/tutorials/graph_api_template.html).

What is a Personal Cloudlet?

A personal Cloudlet is the end-users personal data stored on the OPENi platform. The Cloudlet is built up from the user interataction with OPENi enabled apps. From an end users perspective it is an easy to use free storage facility that affords them full control over the storage and sharing of their personal data. From the app developers perspective it is 1) a rich source of data that they can utilise to enhance their applications, 2) a mechanisms to simplify the process of sharing, and 3) a third party service to which they can outsource the physical, legal, and ethical management of their mobile app's data.

Security considerations.

The OPENi security and privacy framework provides the resource platforms with a unified security, privacy and identity concept. It enables user-centric management to late-bound and dynamic data and APIs, while providing the developer with the ability to access the data in a privacy concerning manner. At its corte the security platform focusses on fine grain access control and single sign on capabilities. It provides the end-user maximum control of their data by enabling centric permission management before and after the initial installation of a client. Additionally it can serve as an identity provider to external applications and services and provides the infrastructure security for the platform as a whole.

Client Libraries

To experiment with OPENi's Graph API, security, and cloudlet frameworks you can use the REST Endpoints HTML UI. For Android application development we provide a mobile client library. This library abstracts and simplify access to the OPENi services, in simple terms it aggregates the REST clients of the three frameworks, but it also adds a significant number of enhancements including a number of security mechanisms and UI components.

We also provide standalone Java/Android client libraries for the individual frameworks:

  • Android client library
  • Security framework Java client library
  • ALL REST Endpoints HTML UI
  • Graph API REST Endpoints HTML UI
  • Cloudlet REST Endpoints HTML UI
  • Security API REST Endpoints HTML UI

Architecture

At a high level the OPENi Platform is composed of four major components: 1) the mobile client library, 2) the security framework, 3) the API framework, and 4) the Cloudlet framework. Each component is developed using a different language and a different application stack as is suited to its functionality, yet all are presented to mobile developers through a single suite unified REST Endpoints. Internally the individual frameworks also interact with each other through their respective REST endpoints and SDK's. All interaction with external cloud-based services is performed by the API framework through it's Cloud-based services (CBS) connectors.
